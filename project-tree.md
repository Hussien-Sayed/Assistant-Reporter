    - multi-agent AI system
        - class for creating multi-agent AI system using LangGraph built speicifcally for this project (news reporter agent)
            - input: expects to get preference keywords from user from a file named "preferences.txt", later on this file will be replaced with a front end interface
            - agent nodes: 
                - web query & fetching data from internet (given a search query, it is able to use the query API and fetch API tools) 
                - Preference creator (uses preference keywords by user to generate relevant search queries based on what the user wants and returns top-k results), where k is configurable via .env file
                - Summarizer (summarizes the fetched data and returns a structured summary for each news page)
                - Image describer (uses the API LLM image class to describe images found in the news pages)
                - Content Checker (checks if the output from the summarizer is correct by constructing a search query, getting the data from different source and validating that the generated content matches the fetched data)
                - News Reporter (uses output from each article generated by the summarizer and content checker to generate a final news report layout. So the main functionality of this agent isn't rewriting but returning a JSON layout desrcibing the width, hight and postion for each article to appear on screen, then the order of the articles when viewed as a simple list, it should consider adding relevant images to the layout and relevnacy can be checked by Image describer agent)



            - behaviour: we need to have an autonomous system that runs on its own to generate a dashboard of news articles. first the preference creator agent will generate search queries based on the user preferences input, then the web query & fetching data from internet agent will fetch the data using the search queries provided by the preference creator agent, then the summarizer agent will summarize the data, then the image describer agent will describe the images found in the news pages and add them to the summary in a know format (eg. <image_description>...</image_description>), then the content checker agent will check the accuracy of the summarizer's output by checking the summary for each article (loop), and then generating a summarized search query from that summary then passing it to the web query & fetching data from internet agent then passing it back to the summarizer agent then double checking the quality by comparing the new summary with the old one (this is a long process that could be divided into smaller agents), if inside the loop the content checker agent finds that the summary is not accurate, it will remove the article from the list of reported articles, and finally the news reporter agent will generate a final news report layout.
            - methods: agentic_system.run()
